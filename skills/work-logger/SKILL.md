# Work Logger

将当前 Codex 会话中完成的工作总结并记录到工作日志。

## 日志存储

所有日志统一存储在 `~/work-logs/` 目录下，按日期命名：

- 文件名格式：`YYYY-MM-DD.md`
- 同一天的多次记录追加到同一个文件中
- 每条记录包含时间戳，按时间顺序排列

## 工作流程

### Step 1: 回顾当前会话

1. 回顾本次会话的完整对话历史，梳理完成了哪些工作
2. 识别关键成果：
   - 创建/修改了哪些文件
   - 解决了什么问题
   - 安装/配置了什么工具
   - 做了哪些调研
   - 产出了什么成果（项目、文档、配置等）

### Step 2: 生成日志内容

按以下模板生成日志条目：

```markdown
## HH:MM - {工作主题简述}

**背景**: 一句话说明为什么做这件事

**完成的工作**:
- 具体工作项 1
- 具体工作项 2
- ...

**关键成果**:
- 成果 1（附文件路径或链接）
- 成果 2
- ...

**相关文件**:
- `path/to/file1` - 说明
- `path/to/file2` - 说明

---
```

### Step 3: 写入日志文件

1. 确定今天的日期，生成文件名 `YYYY-MM-DD.md`
2. 检查文件是否已存在：
   - **已存在** → 追加新条目到文件末尾
   - **不存在** → 创建新文件，先写入日期标题 `# YYYY-MM-DD 工作日志`，再写入条目
3. 写入完成后告知用户日志路径

## 注意事项

- 日志内容应该简洁精炼，突出关键信息，避免冗余
- 文件路径使用相对于工作区的路径
- 如果会话中有多个不同主题的工作，分成多个条目记录
- 不要记录闲聊或无实际产出的对话
- 时间使用 24 小时制
