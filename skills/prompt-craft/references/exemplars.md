## 金标准范例

**`prompts/15_item_score_v26.md`——条目评分（判断类的完整示范）**
九模块全部在位，写任何判断/评分类 prompt 前先通读它。重点看：
- 背景段的职责边界 +「错放伤害重于漏掉」非对称代价声明（模块 1）
- 一句话判断锚，且被 D5 档位表复用为分界检验（模块 2）
- 七步流水线：硬门→否决→分类→打分→补充→自查→输出（模块 3）
- 五维档位表：每档「定义+检验+边界情况+真实样例」四件套；「名气不改变本档」「篇幅不是证据」等把历史错误固化成规程（模块 4）
- 附录四条真实生产样例，覆盖四条判定路径含出局路径的输出形态（模块 5）
- veto 表「先正向出口、再否决条件」句式（模块 6）
- 输入节信任边界段：注入尝试正向路由到 veto 口径（模块 7）
- 分析区固定结构 +「解析器只取最后一个 JSON 块」（模块 8）

## 写作纪律（本仓库约定，通用适用）

1. 头部一行版本号（如 `> 版本：item_score_v26_5_real_samples_2026_07_11`），正文零版本叙事。
2. 八段结构：背景/角色/目标/任务/执行步骤/输入/输出/注意事项；样例放附录。
3. 表格承载档位与枚举（含义/边界/样例分列），不用大段散文。
4. 判断类必须「先分析后输出」；解析器契约写在输出节第一句。
5. 改动生产 prompt 走 worktree + 标注集回归，与代码改动同等对待。
