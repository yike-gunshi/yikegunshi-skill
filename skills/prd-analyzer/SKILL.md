# PRD Analyzer

## Overview

分析产品需求文档（PRD）和界面截图，从产品可用性、评测可信度、工程可落地性三个维度发现问题，输出结构化的专业评审报告。

## Usage

### Input Requirements

用户提供包含以下文件的文件夹路径：
- **PRD 文档**：PDF 格式的产品需求文档
- **产品界面截图**：PNG、JPG、JPEG 格式的设计稿或界面截图

### Workflow

1. **收集文件**
   - 使用 `Glob` 工具查找指定目录下的所有 PDF 文件：`**/*.pdf`
   - 使用 `Glob` 工具查找所有图片文件：`**/*.{png,jpg,jpeg,PNG,JPG,JPEG}`

2. **读取和分析内容**
   - 使用 `Read` 工具读取所有 PDF 文件内容
   - 使用 `mcp__4_5v_mcp__analyze_image` 工具分析每张图片
     - 提示词：`"详细描述这个产品界面的布局结构、功能模块、交互元素和视觉设计风格，以便进行产品评审。"`
   - 将所有内容整合为完整的产品上下文

3. **应用分析框架**
   - 参考 `references/analysis_framework.md` 中的分析框架
   - 基于以下角色进行分析：资深 AI 产品负责人 + 资深交互体验评审 + 搜索评测平台领域专家
   - 从三个维度评审：产品可用性、评测可信度、工程可落地性

4. **生成报告**
   - 严格按照 `references/analysis_framework.md` 中定义的 7 部分结构输出
   - 使用 Markdown 格式
   - 保存到用户指定的输出路径（默认为当前目录的 `PRD_分析报告.md`）

## Output Structure

分析报告必须包含以下 7 个部分：

1. **需求澄清与主链路复述**
   - 背景与目标
   - 核心用户与关键使用场景
   - 主流程（端到端）
   - 隐含假设（必填，含风险等级）

2. **PRD 审阅建议**
   - 每条包含：问题点、为什么是问题、可能后果、建议改法、实现/交付注意事项、优先级

3. **风险清单**
   - 技术风险、产品风险、用户体验风险等

4. **方案升级建议**
   - 3 条"高收益、低改造"优化方案

5. **优先级与行动清单**（表格格式）
   - 包含：优先级、问题/机会点、影响面、建议方案、依赖、工作量、验收方式

6. **待补充信息**
   - 列出必须补充的关键信息及用途

7. **发散思考**
   - 对需求的理解、想法、优化建议、未来演变形态

## Important Notes

- 只基于用户提供的 PRD 文档和截图进行分析，不要假设额外的信息
- 如果 PRD 或截图信息不足，在相应部分明确标注"信息缺口"
- 输出必须严格遵循 7 部分结构，不得省略任何部分
- 风险等级标注：高/中/低
- 优先级标注：P0/P1/P2
- 工作量标注：S（小）/M（中）/L（大）
